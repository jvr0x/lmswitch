# lmswitch

List and toggle local LLMs served on the Spark from per-model env files.

`lmswitch` reads `~/.config/llama-models/*.env`, shows a table of every model
(grouped by family) with its size, download state, port, and whether it's
currently serving, and lets you start/stop them. GGUF models run under
llama.cpp (`llama-server` via the `llama-model@` systemd user unit); safetensors
models run under vLLM in Docker (`vllm-model@` unit). It also regenerates the
opencode provider list from whatever is currently serving.

## Usage

```
lmswitch                     # interactive: show the table, then type model #s to toggle
lmswitch list                # just print the table (read-only)
lmswitch on  <name|#>        # start a model
lmswitch off <name|#>        # stop a model
lmswitch sync                # regenerate opencode.json from currently-serving models
```

In the interactive prompt you can toggle several at once, space/comma/dash
separated: `8 9 24`.

## Install

The script lives here; `~/.local/bin/lmswitch` is a symlink to it (so it stays
on `$PATH`):

```
ln -sf ~/utils/lmswitch/lmswitch ~/.local/bin/lmswitch
```

## Model env files (`~/.config/llama-models/<name>.env`)

```
LLAMA_MODEL=<path under ~/models>      # GGUF file, or dir for vLLM
LLAMA_PORT=8089
LLAMA_CTX=262144
LLAMA_DISPLAY_NAME="Qwen3.6-35B-A3B"
# LLAMA_MMPROJ=<path>                  # vision projector (optional)
# RUNTIME=vllm                         # safetensors via vLLM instead of llama.cpp
# GPU_MEM_UTIL=0.55                    # vLLM only
# VLLM_TOOL_PARSER=qwen3_coder         # vLLM tool-call parser
# VLLM_REASONING_PARSER=qwen3          # vLLM reasoning parser
```

## Dependencies (not in this repo)

- `~/.local/bin/llama-server-start`, `~/.local/bin/vllm-server-start` — launchers
- `~/.config/systemd/user/llama-model@.service`, `vllm-model@.service` — units
- `~/.config/opencode/opencode.json` — regenerated by `lmswitch sync`
